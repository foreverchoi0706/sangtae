상태 관리 라이브러리 sangtae

atom 개념만으로 전역 상태, 파생 상태, 비동기 상태를 모두 다룰 수 있는 초경량 상태 관리 라이브러리. React, Vue 어디서든 동일한 API로 동작합니다.

시작하기 GitHub에서 보기
$ npm install sangtae-js

Features

필요한 것만, 딱 그만큼

복잡한 보일러플레이트 없이 atom 하나로 상태를 선언하고 어디서든 구독하세요.

⚛️

Atom 기반 설계

상태를 최소 단위인 atom으로 격리합니다. Provider나 전역 store 없이 atom 단위로 구독자를 관리합니다.

🔗

파생 Atom

createDerivedAtom으로 여러 atom을 조합해 자동으로 추적되는 계산된 상태를 만듭니다.

⏳

비동기 Atom + Suspense

Promise 기반 비동기 상태를 createAsyncAtom으로 감싸면 React/Vue Suspense와 자연스럽게 연동됩니다.

🧹

GC-Friendly 메모리 관리

마지막 구독자가 해제되면 내부 listener Set을 null로 되돌려 메모리 누수 없이 GC 대상이 됩니다.

🔒

TypeScript 우선

제네릭 기반 API로 컴파일 타임 타입을 보장합니다. atom 타입이 그대로 추론됩니다.

🌐

Framework Agnostic

React, Vue 없이도 get · set · subscribe 만으로 어떤 환경에서든 동작합니다.

API Reference

심플한 API, 강력한 표현력

6개의 함수만으로 복잡한 상태 시나리오를 모두 처리할 수 있습니다.

import { createAtom, get, set, subscribe } from "sangtae-js";

// atom 생성
const counterAtom = createAtom(0);

// 구독 — 값이 변경될 때마다 콜백 실행
const unsubscribe = subscribe(counterAtom, (value) => {
  console.log("현재 값:", value);
});

set(counterAtom, 1);  // 콘솔: "현재 값: 1"
console.log(get(counterAtom));  // 1

// 동일 값으로 set하면 구독자에게 알리지 않음
set(counterAtom, 1);  // 콜백 실행 안 됨 (Object.is 비교)

unsubscribe();  // 구독 해제 — 마지막 구독자면 내부 Set도 GC 대상
import { createAtom, createDerivedAtom, set, get } from "sangtae-js";

const priceAtom    = createAtom(10_000);
const quantityAtom = createAtom(2);

// 의존 atom을 자동으로 추적
const totalPriceAtom = createDerivedAtom((get) =>
  get(priceAtom) * get(quantityAtom)
);

console.log(get(totalPriceAtom));  // 20000

set(quantityAtom, 3);
console.log(get(totalPriceAtom));  // 30000 (자동 갱신)

// 파생 값이 같으면 구독자에게 알리지 않음
set(priceAtom, 5_000);
set(quantityAtom, 6);  // total은 여전히 30000 → 알림 없음
import { createAsyncAtom, getAsync, set } from "sangtae-js";

// Promise로 비동기 atom 생성
const $todos = createAsyncAtom(
  fetch("/api/todos").then(res => res.json())
);

// await로 값을 가져올 수 있음
const todos = await getAsync($todos);
console.log(todos);

// 동일 Promise로 createAsyncAtom을 다시 호출해도
// WeakMap 캐시 덕분에 항상 같은 atom 인스턴스를 반환

// set으로 값을 동기적으로 덮어쓸 수 있음
set($todos, [...todos, { id: 99, title: "새 할 일", completed: false }]);
// stores/counter.ts
import { createAtom } from "sangtae-js";

export const $counter = createAtom(0);
export const $step    = createAtom(1);

// components/Counter.tsx
import { useAtom } from "sangtae-react";
import { $counter, $step } from "../stores/counter";

export default function Counter() {
  const [counter, setCounter] = useAtom($counter);
  const [step]              = useAtom($step);

  return (
    <div>
      <button onClick={() => setCounter(counter - step)}>-</button>
      <strong>{counter}</strong>
      <button onClick={() => setCounter(counter + step)}>+</button>
    </div>
  );
}
<!-- components/CounterCard.vue -->
<script setup lang="ts">
import { computed } from "vue";
import { useAtom } from "sangtae-vue";
import { $counter, $step } from "@/stores/counter";

const [counter, setCounter] = useAtom($counter);
const [step]              = useAtom($step);

const decrease = () => setCounter(counter.value - step.value);
const increase = () => setCounter(counter.value + step.value);
const label    = computed(() => `${counter.value} (step ${step.value})`);
</script>

<template>
  <section>
    <p>{{ label }}</p>
    <button @click="decrease">-</button>
    <button @click="increase">+</button>
  </section>
</template>

Architecture

내부 설계 원칙

Symbol 기반 은닉 필드, Object.is 비교, WeakMap 캐시로 성능과 안전성을 동시에 확보했습니다.

Atom 내부 구조

Symbol(VALUE) 현재 값을 Symbol 키로 은닉하여 외부에서 직접 접근 불가
Symbol(LISTENERS) 구독자 Set. 마지막 구독 해제 시 null로 되돌려 GC 대상화
Object.is NaN, ±0 등 엣지케이스도 정확히 처리하는 동등성 비교
Readonly<T> 외부에 불변으로 노출 — API 함수로만 읽기·쓰기 가능

파생 Atom 동작

trackedGet callback(get) 실행 중 접근한 atom을 Set에 자동 수집
Map 기반 구독 의존 atom 변경 시 끊어진 구독은 즉시 해제
Suspense 대응 평가 중 Promise throw → PENDING, 해결 시 자동 재평가
순환 차단 파생 atom이 자신을 참조하는 무한 루프 구조적으로 방지

비동기 Atom 캐시

WeakMap Promise를 키로 캐싱 — 동일 Promise면 항상 같은 atom 반환
Proxy get 트랩 PENDING이면 Promise throw, ERROR면 에러 throw, SUCCESS면 데이터 반환
getAsync throw된 Promise를 await 처리해 async/await 패턴으로 사용 가능
set 오버라이드 동기값으로 덮어쓰면 status가 success로 전환

React 연동 설계

useSyncExternalStore React 18 Concurrent 렌더링에서도 스냅샷 싱크 안전 보장
useCallback atom props 변경 시 구독/해제 반복 방지를 위해 안정화
배치 업데이트 React의 자동 배치(Auto Batching)와 연동하여 렌더 횟수 최소화
onUnmounted Vue에서는 생명주기 훅으로 구독 해제 자동 관리

Packages

3개의 패키지, 하나의 API

npm workspace 기반 모노레포로 구성되어 있으며, 각 패키지는 독립적으로 설치해 사용할 수 있습니다.

⚡
sangtae-js

프레임워크 독립 코어 라이브러리. atom 생성, 파생, 비동기 상태, 구독 관리를 포함한 모든 핵심 기능을 담고 있습니다.

createAtom createDerivedAtom createAsyncAtom Vanilla JS
⚛️
sangtae-react

React 18+ 전용 훅 바인딩. useSyncExternalStore 기반의 useAtom 훅으로 Concurrent 렌더링에서도 안전하게 동작합니다.

useAtom React 18+ Suspense
💚
sangtae-vue

Vue 3 Composition API 전용 훅 바인딩. Ref 기반 인터페이스와 onUnmounted 생명주기 훅으로 자동 구독 관리를 제공합니다.

핵심 API부터 선택 기능, 프레임워크 연동, 샘플 앱까지 모두 완료되었습니다.

분류 항목 상태 비고

파생 Atom createDerivedAtom 완료 의존 atom 자동 추적, 변경 시 재계산
비동기 Atom createAsyncAtom · getAsync 완료 Suspense 대응, WeakMap 캐싱

메모리 관리 Listener GC 처리 완료 마지막 구독 해지 시 Set → null
배치 업데이트 React Auto Batching 연동 완료 별도 배치 API 없이 React와 연동
단위 테스트 @sangtae-js Vitest 완료 필수/선택 기능 전체 커버

Quick Start

30초 안에 시작하기

npm workspace 기반 모노레포로 구성되어 있습니다. 전체 환경을 한 번에 설치하고 실행하세요.

# 1. 의존성 설치 (워크스페이스 전체)
npm run install:all

# 2. sangtae-js, sangtae-react, sangtae-vue 빌드
npm run build:all
# React + Vite 데모 앱 실행
npm run preview:sample-app-react

# → http://localhost:4173 에서 확인
# 카운터, 파생 상태, 비동기 상태, 수동 구독 시나리오 포함
# Vue 3 + Vite 데모 앱 실행
npm run preview:sample-app-vue

# → http://localhost:4174 에서 확인
# React 버전과 동일한 시나리오를 Vue 3 컴포넌트로 제공
# @sangtae-js 단위 테스트 (watch mode)
npm run test --workspace=@sangtae-js

# CI 환경 단발 실행
npm run test:run --workspace=@sangtae-js
